## 引言

7 月 5 日，Monad Builder Camp 正式开课。其实在 [上一篇文章](/blog/web3-notes-01) 里，我已经用 Web2 的视角把钱包、区块链、智能合约和 DApp 这套结构粗粗过了一遍。但「看过地图」和「真正踏上那条路」完全是两回事——Day 1 的任务，就是让我从理论走向操作。

今天的重点不是赚测试币、也不是冲某个 DApp，而是两件事：**建立安全的钱包使用习惯**，以及**建立最基本的链上认知**。我按课程要求，完成了钱包准备、网络配置、区块浏览器查询，以及第一笔 Monad Testnet 链上交易。下面是我的完整记录。

## 一、课程专用钱包：为什么要单独准备一个？

课程第一条规定写得很清楚：**创建或准备一个课程专用钱包，不要使用主力钱包。**

这一点我一开始觉得「是不是有点小题大做」，但仔细想了一下，这和 Web2 开发里「测试环境和生产环境分离」其实是同一个逻辑。你在本地跑 `localhost:3000` 的时候，不会拿线上数据库去试；进链上世界，也不该拿存有真实资产的主钱包去练手。

### 我使用的钱包工具

我使用的是 **Rabby Wallet**。之前用过 MetaMask，但 Rabby 在多链切换和 DApp 连接上更顺手一些，界面也把交易预览做得比较清楚——对新手来说，「发起转账之前先看一眼 from / to / value」这件事很重要。不管用哪款钱包，原理都一样：帮你生成一对密钥，管理你的链上地址。

### 操作步骤

1. 在 Rabby 里新建一个账户，专门用于本次课程。
2. **妥善备份助记词**，存到安全的地方——这一步和「备份数据库」一样重要，丢了就真的找不回来。
3. 给这个钱包起个容易识别的名字，比如「Monad Camp 专用」，避免和日常钱包搞混。

### 我的课程钱包地址

```
0x86E15734C08423AE27bd093F9564d479C26F2156
```

> 提醒：地址可以公开，但**助记词和私钥绝对不要出现在任何文章、截图或聊天记录里**。

## 二、添加 Monad Testnet 网络

钱包有了，下一步是让它能「说话」——连接到 Monad 的测试网络。

### 网络配置信息

| 字段 | 值 |
| --- | --- |
| 网络名称 | Monad Testnet |
| RPC URL | `https://testnet-rpc.monad.xyz` |
| Chain ID | `10143` |
| 货币符号 | MON |
| 区块浏览器 | `https://testnet.monadexplorer.com` |

在 Rabby 里：**设置 → 自定义网络 → 添加自定义网络**，把上面的信息填进去即可。也可以直接访问 [Monad Testnet 应用中心](https://testnet.monad.xyz/)，通常有一键添加网络的按钮，比手动填省事。

### 添加结果

切换网络后，Rabby 顶部显示 **Monad Testnet**，状态正常。✅ 网络添加成功。

## 三、第一次打开区块浏览器

配置完网络，课程要求打开 **Monad Explorer**（或相关区块浏览器），确认可以查询地址。

我把课程钱包地址粘贴进 [testnet.monadexplorer.com](https://testnet.monadexplorer.com) 的搜索框，第一次真正「看见」链上的自己。

### 我看到的页面信息

- **Address**：我的钱包地址，一串 `0x` 开头的字符。
- **Balance**：当前账户余额，刚创建时应该是 `0 MON`。
- **Transactions**：历史交易列表，空钱包时这里是空的。
- 页面上还有一些概览数据，比如该地址的首次出现时间、交易笔数等。

说实话，第一次打开区块浏览器的感觉，有点像第一次用 `git log` 看提交历史——**所有操作都会被公开、永久地记录下来**。没有「后台悄悄改数据」这一说，每一笔变动都摆在那里，谁都可以查。

## 四、链上产品和普通互联网产品有什么不同？

课程要求用自己的话简单说明。以下是我 Day 1 的初步理解，可能还不完整，但已经比「去中心化」四个字具体多了。

### 普通互联网产品（Web2）

> **用户请求 → 平台服务器处理 → 平台数据库改状态 → 返回结果**

用户看到的是界面和功能，但真正的数据和规则藏在平台的服务器里。你信任的是「这个平台不会乱来」——淘宝说你的订单已发货，你就相信它；微信说消息已读，你也只能相信它。出了问题，往往只能找客服、申诉、等人工处理。

### 链上产品（Web3）

> **用户签名 → 交易广播到网络 → 节点验证并执行 → 状态写入区块链 → 所有人可验证**

几个关键差异：

1. **没有「平台替你记账」这一层。** 状态直接写在链上，改一条记录需要全网共识，不是某个管理员在后台点一下就能改的。
2. **身份是钱包，不是账号密码。** 你不是「登录」系统，而是用私钥签名证明「这个地址归我管」。
3. **操作是交易，不是 API 调用。** 每一次转账、每一次和合约交互，都是一笔需要付费（Gas）、需要等待确认、会被永久记录的操作。
4. **透明与不可逆。** 区块浏览器上谁都能查，成功了就是成功了，失败了也可能留下一条失败记录——这和 Web2 里「操作失败就当没发生过」很不一样。

如果用之前文章里的「小镇公告栏」类比：Web2 是你把纸条交给镇长，由他决定是否贴、贴什么、能不能撕；Web3 是你自己把纸条贴到公告栏上，全镇居民一起公证，贴上去就改不了。

## 五、第一笔 Monad Testnet 链上交易

第二部分任务，是完成真正的链上操作：拿测试币、转账、保存交易哈希、在浏览器里复盘。

### 1. 获取测试网资产

测试网上的 MON **没有真实价值**，纯粹用来练手。那从哪里领呢？Web3 里有个很形象的叫法：**Faucet**，中文常译作「水龙头」——就像公共卫生间的水龙头，打开会滴下来一点免费的水；测试网水龙头也是类似逻辑：**你提交钱包地址，它会给你发一点测试代币**，额度有限、领完可能要等冷却时间。

所以「官方水龙头」并不是说有个真的水龙头，而是指 **Monad 团队提供的官方测试币领取页面**（[faucet.monad.xyz](https://faucet.monad.xyz/)）。和第三方水龙头相比，官方的一般更稳定、不容易遇到恶意网站。

我通过这个页面领取了测试 MON，流程大致是：

1. 打开领取页面，连接课程专用钱包。
2. 输入钱包地址（或已连接则自动识别）。
3. 完成验证（可能有验证码或社交验证），提交领取请求。
4. 等待几秒到几分钟，Rabby 里余额从 `0` 变成有数了。

领到测试币的那一刻，我第一次感觉到：链上的「钱」不是数据库里一个数字字段，而是**这个地址在链上状态里的真实余额**。

### 2. 完成一次测试网转账

我给自己准备的另一个测试地址（同样是课程专用，不是主力钱包）转了一笔 MON。

- **From**：我的课程钱包 `0x86E15734C08423AE27bd093F9564d479C26F2156`
- **To**：我的第二个测试地址 `0x240e0b16297C9F738260866B0934fb99CBC33e7E`
- **Amount**：`4 MON`

在 Rabby 里确认交易后，会弹出一个 **Transaction Hash（交易哈希）**，我把它保存了下来：


## 六、用自己的话解释：这笔交易到底发生了什么？

### From / To 分别是谁？

- **From** 是「发起者」——也就是我，用课程钱包的私钥签名，授权从自己的账户里划走 4 MON。
- **To** 是「接收者」——我的第二个测试地址。注意，这里 To 可以是一个普通钱包地址，也可以是一个智能合约地址；Day 1 我做的是最简单的钱包到钱包转账。

### Value 表示什么？

**Value** 是这笔交易转移的原生代币数量，单位是 MON。它不包括 Gas 费——Gas 是另外从 From 地址扣的「过路费」，用来补偿网络节点帮你打包、验证这笔交易。

可以把它理解成：Value 是你寄出去的「包裹内容」，Gas 是你付给快递员的「运费」。

### Gas 和手续费代表什么？

每一笔链上操作都需要计算资源。网络里的验证节点要检查你的签名是否合法、余额是否足够、交易格式是否正确，然后把它写进区块。这些工作不是免费的，所以你要付 **Gas 费**。

- **Gas Limit**：你愿意为这笔交易支付的最大 Gas 上限（类似「最多愿意出多少运费」）。
- **Gas Price / Gas Fee**：实际单价和总费用。
- **Transaction Fee** = Gas Used × Gas Price，从 From 地址扣除。

我这笔测试网手续费是 0.0038556 MON，数额很小，但**不是零**——链上操作从第一笔开始就真实存在成本。放到主网上，同样的操作可能就要几毛到几块不等，所以「每笔交易都要付 Gas」是需要尽早建立的习惯。

### 交易状态：成功还是失败？

我这笔是 **Success**。意味着：签名有效、余额足够、网络共识通过，4 MON 已经从 From 划到了 To，链上状态已更新。

### 失败的交易为什么也可能消耗 Gas？

这是 Day 1 里我觉得反直觉、但又很重要的一点：

> **Gas 付的是「执行计算」的费用，不是「交易成功」的奖励。**

就算你的交易因为余额不足、合约 revert、Gas 给少了等原因失败了，节点依然替你跑完了验证和执行流程——这部分算力已经消耗了，所以 Gas 照样扣。就像你叫外卖，骑手到了门口发现你填错地址，餐送不进去，但配送费通常还是要付的。

## 七、Day 1 小结

今天没有写一行智能合约，也没有部署任何 DApp，但我感觉比「直接冲进去写代码」踏实得多：

1. **钱包隔离**：课程专用钱包已就绪，主力钱包 untouched。
2. **网络就绪**：Monad Testnet 添加成功，能正常切换。
3. **浏览器会用**：能查地址、能查交易、能读懂基本字段。
4. **第一笔交易**：领水 → 转账 → 存 hash → 复盘，完整走通一遍。
5. **认知更新**：链上产品不是「多了一个区块链字段的 App」，而是**状态在链上、规则靠共识、操作靠签名**的另一套系统。

如果 Day 0 是在纸上画地图，那 Day 1 就是第一次系鞋带出门跑步。路还很长，但至少我知道钱包该放哪只口袋、区块浏览器该往哪看了。
