## 引言

我把自己的博客从 WordPress 完整迁移到了 Astro。如果只是从技术角度来看，这其实算不上什么特别复杂的项目。无非就是重新搭建框架、迁移文章、设计新的样式、配置 Cloudflare Pages、重新部署，以及修复迁移过程中各种各样的小 Bug。

其实我也有想过，为什么要这样折腾？WordPress 没有什么不好啊。成熟、稳定、生态丰富，几乎所有博客需要的功能，它都有现成的插件。那我为什么还要重构，然后使用 Astro 呢？

很简单，博客，不应该只是文章的容器。

我一直觉得，技术博客最重要的不是文章，而是让它应该能够随着自己的成长一起成长。我今天学 Astro，博客就应该可以尝试 Astro。我明天研究 AI，博客也应该能够自然地接入 AI。博客本身，应该成为自己的实验场，而不是一个单纯的、已经固定好的 CMS。

如果每一次新的尝试，都要先思考："这个插件支不支持？"那它已经开始限制我的学习了。

![](/images/blog/06/image-1.jpg)

这是我之前的 WordPress 主题。我个人是一个比较喜欢创造设计的人，包括这个可能略显简约的 WordPress 主题，其实就是我自行设计搭建的。

在一次偶然的机会下，我看到了一个用 Astro 框架设计的博客，说实话确实让我眼前一亮。相比于"开箱即用的成品家具"，这种"自己组装的乐高积木"明显更加让我心动。于是我果断地舍弃掉了 WordPress。你已经不被需要了，WordPress（无慈悲）。

接下来在搭建 Astro 框架时，我突然想到了一个新的问题：到底为什么要拥有一个属于自己的网站？

我不知道这个问题是怎么出现在我脑中的，可能是在迁移数据的时候，也可能是在设计 UI 的时候……总之它就这么突兀地出现在了我脑中，也促使我写下了这篇文章。

## 博客到底要做什么

之前在知乎上有这样一个问题：现在这个时代，写个人博客还有意义吗？

这是我很久之前看到的一个问题。当时我还没有写博客的习惯，所以我当时的回答是：没有意义，或者说意义不大。毕竟现在不管是知乎、小红书还是 X，亦或者是一些专业的技术论坛，比如掘金、CSDN 这些，它们不管是功能还是设计都可以说是十分的完美，并且天生就带着庞大的流量。为什么不直接在这些平台上分享呢？

但是现在我的想法改变了。

从我开始搭建自己的博客的时候，从我真正开始维护它以后，我发现自己越来越少关注阅读量，而越来越关注"这篇文章，未来的我还会不会回来再看？""这篇文章有没有让我有所成长？"

我意识到那些平台都有一个共同特点，那就是它们不属于我。我的内容发在这些平台上面，它们的算法决定了观看人群，它们的规则约束了范围。但是博客不一样，博客不是为了流量，博客也没有算法。它只是一个纯粹的记录空间，静静地放在那里，等未来的某一天，我或者别人，偶然走进来。

如果说博客是你的房子，那么社交平台更像是广场。我们在广场上可以聊天，但没人会把全部家当放在广场上。

我想这就是博客的作用。它给了我一个独属于我的互联网空间，在这里我可以记录我想要记住的东西，可以让我把零散的思考，变成长期存在的东西。或许以后会有人偶然走进来，他会看到我曾经踩过的坑，也会看到我曾经犯过的错误。但我想，这些都不是博客最重要的意义。

真正重要的是，多年以后，当我再次回顾的时候，我还能看见曾经那个正在成长的自己。

## 人为什么要记录自己

真正值得被保存的，从来都不是那些知识本身。

在如今这个时代，技术更新得太快了。前几年还流行的框架，如今已经逐渐被新的工具替代。今天写下的某些最佳实践，也许几年以后就已经过时。就像我之前很多时间学习了解 WordPress，如今又因为 Astro 而推倒重来。知识始终会随着时代不断迭代，它们很少有什么东西能够永远正确。

可当我重新翻看自己过去写下的文章时，我发现自己真正感兴趣的，并不是当时学会了什么，而是当时为什么会去学习它。我会看到那时困扰我的问题，会看到当时解决 Bug 时的喜悦，也会看到那些现在看来有些稚嫩、甚至有些天真的思考。

![](/images/blog/06/image-2.png)

这是我很久之前给"我"写的一封信，19 年的我还是初一。我重新看到这封邮件的时候其实愣了很久，因为我完全忘记了。这封七年前的、"我"一个字一个字敲出来的邮件，其实回看一下内容真的很幼稚。但这也是我想要说的：记录下来的，不只是知识，更是一个人在某个时间点真实的样子。

人其实是一种很容易遗忘的生物。因为害怕遗忘，所以很多人都会记录。有人写朋友圈，有人发微博，有人写知乎，有人拍照片。它们形式不同，但是本质都一样——我们总是在害怕某些东西消失，于是不断留下痕迹。

因为时间总是比想象中更擅长冲刷细节。或许几年以后，我们还能记得发生过这件事，却已经忘记了当时为什么激动、为什么犹豫、为什么坚持。那些情绪像退潮后的海水，只留下一个模糊的轮廓。

所以在这个时间点，在这个我即将从我的学生生涯迈向社会的节点，我觉得我要记录些什么。不管是我现如今的学习经历，还是说只是日常的行为思考，我都想要记住。不仅是为了向互联网证明什么，而是为了给未来的自己留下一个能够回来的坐标。

希望当未来我再次打开这篇文章的时候，我看到的不只是几千个汉字，而是那个凌晨两三点还坐在电脑前、认真思考、认真记录的自己。

## 所以，我为什么选择 Astro

回过头来看，我发现自己真正迁移的，其实不是博客，而是自己对于博客的理解。

一开始，我只是希望拥有一个可以写文章的网站。后来，我开始希望它能够按照自己的想法去设计。再后来，我希望它能够加入新的交互、尝试新的技术。博客对我来说已经不只是一个放文章的地方，而更像是一个和我一起成长的东西。

或许很多年以后，Astro 也会像 WordPress 一样，成为我曾经使用过的一个框架。也许那个时候，我会把整个博客迁移到另一个新的技术栈；也许今天我自认为不错的 UI 设计，也会显得有些过时。但我希望，有一样东西不要变——那就是我依然会不断学习新的东西，不断记录新的思考，也依然会在某个深夜，坐在电脑前，把那些转瞬即逝的灵感写下来。

博客会迁移，框架会迭代，设计会重构。但真正一直在更新的，应该是屏幕前的自己。
